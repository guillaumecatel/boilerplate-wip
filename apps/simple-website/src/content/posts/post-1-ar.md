# مقالتي في المدونة

مقالتي الأولى في المدونة

## الصيغة الأساسية

Markdown هي صيغة خفيفة وسهلة الاستخدام لتنسيق كتاباتك.

### العناوين

عندما يكون محتوى المقالة واسعًا، يمكنك استخدام العناوين للتقسيم:

```markdown
# عنوان 1

## عنوان 2

## عنوان كبير

### عنوان صغير
```

معاينات العناوين ستعطل بنية المقالة، لذا لا يتم عرضها هنا.

### الخط العريض والمائل

```markdown
_نص مائل_ و **نص عريض**، معًا سيكون **_نص عريض مائل_**
```

المعاينة:

_نص مائل_ و **نص عريض**، معًا سيكون **_نص عريض مائل_**

### الروابط

```markdown
رابط نصي [اسم الرابط](http://link-url)
```

المعاينة:

رابط نصي [اسم الرابط](http://link-url)

### الكود المضمن

```markdown
هذا `كود مضمن`
```

المعاينة:

هذا `كود مضمن`

### كتل الكود

````markdown
```js
// حساب فيبوناتشي
function fibonacci(n) {
  if (n <= 1) return 1
  const result = fibonacci(n - 1) + fibonacci(n - 2) // [\!code --]
  return fibonacci(n - 1) + fibonacci(n - 2) // [\!code ++]
}
```
````

المعاينة:

```js
// حساب فيبوناتشي
function fibonacci(n) {
  if (n <= 1) return 1
  const result = fibonacci(n - 1) + fibonacci(n - 2) // [!code --]
  return fibonacci(n - 1) + fibonacci(n - 2) // [!code ++]
}
```

يستخدم حاليًا shiki كإضافة لتمييز الكود. للغات المدعومة، راجع [Shiki: Languages](https://shiki.matsu.io/languages.html).

### معادلة مضمنة

```markdown

```

المعاينة:

هذه معادلة مضمنة $e^{i\pi} + 1 = 0$

### كتل المعادلات

```markdown
$$
\hat{f}(\xi) = \int_{-\infty}^{\infty} f(x) e^{-2\pi i x \xi} \, dx
$$
```

المعاينة:

$$
\hat{f}(\xi) = \int_{-\infty}^{\infty} f(x) e^{-2\pi i x \xi} \, dx

يستخدم حاليًا KaTeX كإضافة للمعادلات الرياضية. للصيغة المدعومة، راجع [KaTeX Supported Functions](https://katex.org/docs/supported.html).

### الاقتباسات

```markdown
> طلقة نارية، رعد، سيف يرتفع. مشهد من الزهور والدماء.
```

المعاينة:

> طلقة نارية، رعد، سيف يرتفع. مشهد من الزهور والدماء.

يمكنك الاستمرار في تضمين الصيغة داخل الاقتباسات.

### فواصل الأسطر

يحتاج Markdown إلى سطر فارغ لفصل الفقرات.

```markdown
إذا لم تترك سطرًا فارغًا
سيكون في فقرة واحدة

الفقرة الأولى

الفقرة الثانية
```

المعاينة:

إذا لم تترك سطرًا فارغًا
سيكون في فقرة واحدة

الفقرة الأولى

الفقرة الثانية

### الفواصل

إذا كان لديك عادة كتابة الفواصل، يمكنك بدء سطر جديد وإدخال ثلاثة شرطات `---` أو نجوم `***`. اترك سطرًا فارغًا قبل وبعد عندما تكون هناك فقرات:

```markdown
---
```

المعاينة:

---

## التقنيات المتقدمة

### عناصر HTML المضمنة

حاليًا، يتم دعم بعض عناصر HTML المضمنة فقط، بما في ذلك `<kdb> <b> <i> <em> <sup> <sub> <br>`، مثل

#### عرض المفاتيح

```markdown
استخدم <kbd>Ctrl</kbd> + <kbd>Alt</kbd> + <kbd>Del</kbd> لإعادة تشغيل الكمبيوتر
```

المعاينة:

استخدم <kbd>Ctrl</kbd> + <kbd>Alt</kbd> + <kbd>Del</kbd> لإعادة تشغيل الكمبيوتر

#### الخط العريض المائل

```markdown
<b> ينطبق Markdown أيضًا هنا، مثل _عريض_ </b>
```

المعاينة:

<b> ينطبق Markdown أيضًا هنا، مثل _عريض_ </b>

### كتابات HTML أخرى

#### الكتل القابلة للطي

```markdown

```

المعاينة:

<details><summary>انقر للتوسيع</summary>إنه مخفي</details>

| العنوان1 | العنوان2 |
| -------- | -------- |
| المحتوى1 | المحتوى2 |

### الحواشي السفلية

```markdown
استخدم [^حاشية] لإضافة حاشية سفلية عند نقطة المرجع.

ثم، في نهاية المستند، أضف محتوى الحاشية السفلية (سيتم عرضها في نهاية المقالة افتراضيًا).

[^حاشية]: هنا محتوى الحاشية السفلية
```

المعاينة:

استخدم [^حاشية] لإضافة حاشية سفلية عند نقطة المرجع.

ثم، في نهاية المستند، أضف محتوى الحاشية السفلية (سيتم عرضها في نهاية المقالة افتراضيًا).

[^حاشية]: هنا محتوى الحاشية السفلية

### قوائم المهام

```markdown
- [ ] مهمة غير مكتملة
- [x] مهمة مكتملة
```

المعاينة:

- [ ] مهمة غير مكتملة
- [x] مهمة مكتملة

### إيقاف الرموز

إذا كنت بحاجة إلى استخدام رموز markdown مثل \_ # \* في وصفك ولكن لا تريد إيقافها، يمكنك إضافة شرطة مائلة عكسية قبل هذه الرموز، مثل `\_` `\#` `\*` لتجنب ذلك.

```markdown
\_لا أريد أن يكون النص هنا مائلاً\_

\*\*لا أريد أن يكون النص هنا عريضًا\*\*
```

المعاينة:

\_لا أريد أن يكون النص هنا مائلاً\_

\*\*لا أريد أن يكون النص هنا عريضًا\*\*
